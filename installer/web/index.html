Установка 4 MCP-серверов (payfin-kb, payfin-code, gitlab, atlassian) и 10 скиллов для Claude Code — одной командой, ~2 минуты.

curl -fsSL https://payfin-installer-production.up.railway.app/setup.sh | bash

Что делает скрипт

  1. Проверяет Node 20+, git, Claude Code CLI, uv (устанавливает сам если нет)
  2. Проверяет доступ к git.ipoint.uz (VPN)
  3. Спрашивает 4 токена + роль
  4. Удаляет старые регистрации MCP (чтобы не дублировались)
  5. Регистрирует 4 MCP в user scope (работают в любой папке)
  6. Скачивает все скиллы с KB-сервера в ~/.claude/skills/

Перед запуском: собери 4 токена

KB_TOKEN

Формат pfk_xxx. Получить у администратора (@Sardorbek в Telegram).

GITLAB_TOKEN

  1. git.ipoint.uz → правый верх → Edit profile
  2. Слева: Access Tokens → Add new token
  3. Scopes: read_api, read_repository
  4. Create → скопировать (покажется один раз!)

CONFLUENCE_PAT

  1. wiki.ipoint.uz → аватар → Profile
  2. Personal Access Tokens → Create token → скопировать

JIRA_PAT

  1. jira.ipoint.uz → аватар → Profile
  2. Personal Access Tokens → Create token → скопировать

Скрипт попросит AGENT_ROLE. Роль определяет какие папки MCP разрешает читать и какой скилл у тебя «основной».

После установки

  1. Перезапусти Claude Code (закрой полностью и открой снова)
  2. В любой папке выполни: claude mcp list — должны быть 4 сервера:
    • payfin-kb
    • payfin-code
    • gitlab
    • atlassian
  3. В чате спроси «какие слэш-команды доступны» — должны быть /spec, /epic, /implement, /review, /test, /on-task-done, /enrich, /update, /update-tokens, /where-used
  4. Тест: «Покажи composer.json из broker-api» → должен вернуть содержимое через payfin-code

Обновления скиллов в будущем

На KB-сервере может появиться новый скилл или обновление. Чтобы подтянуть — в любом чате:

/update

Ручная установка (если скрипт не подходит)

Развернуть — 4 команды

Эти команды выполняют то же, что делает скрипт. Подставь свои токены.

1. payfin-kb

claude mcp add -s user --transport http payfin-kb \
  https://practical-generosity-production-cb1c.up.railway.app/mcp \
  --header "Authorization: Bearer pfk_..."

2. payfin-code

claude mcp add -s user payfin-code \
  -e GITLAB_URL=https://git.ipoint.uz \
  -e GITLAB_TOKEN=glpat_... \
  -e REPOS=broker-api:broker/backend/broker-api,nasiya-api:nasiya/backend/nasiya-api \
  -e AGENT_ROLE=developer \
  -- npx -y payfin-mcp-code@beta

3. gitlab

claude mcp add -s user gitlab \
  -e GITLAB_API_URL=https://git.ipoint.uz/api/v4 \
  -e GITLAB_PERSONAL_ACCESS_TOKEN=glpat_... \
  -- npx -y @zereight/mcp-gitlab

4. atlassian

На Mac путь к uvx: $HOME/.local/bin/uvx. На Windows: %USERPROFILE%\.local\bin\uvx.exe

claude mcp add -s user atlassian \
  -e CONFLUENCE_URL=https://wiki.ipoint.uz \
  -e JIRA_URL=https://jira.ipoint.uz \
  -e CONFLUENCE_PERSONAL_TOKEN=... \
  -e JIRA_PERSONAL_TOKEN=... \
  -- $HOME/.local/bin/uvx mcp-atlassian

5. Скачать скиллы

curl -sfL "https://practical-generosity-production-cb1c.up.railway.app/setup/skills/update?token=pfk_..." \
  -o ~/.claude/skills/update/SKILL.md

Потом в Claude Code: /update — подтянет остальные.

Если что-то пошло не так

command not found: claude

Claude Code CLI не установлен. Установи: npm i -g @anthropic-ai/claude-code

uvx not found после установки — Atlassian MCP не работает

Claude Code не видит uvx в PATH. Наш скрипт уже должен прописывать абсолютный путь. Если нет — перерегистрируй руками с абсолютным путём:

claude mcp remove -s user atlassian
claude mcp add -s user atlassian -e ... -- $HOME/.local/bin/uvx mcp-atlassian
VPN check упал — «git.ipoint.uz недоступен»

Подключи VPN и повтори запуск. Или ответь y на вопрос «Продолжить без VPN?» если уверен.

payfin-code отвечает «Repo not cloned yet»

Первый запрос запускает клонирование репо на твою машину (в ~/.payfin-code/repos на Mac или %APPDATA%\payfin-code\repos на Windows). Подожди 30 сек и повтори. Для пула обновлений без перезапуска: /* ... */ code_refresh(repo).

PowerShell: running scripts is disabled on this system

Перед irm:

Set-ExecutionPolicy -Scope Process Bypass -Force
MCP появился в claude mcp list, но не работает в чатах

Перезапусти Claude Code полностью (не просто новую сессию — закрой приложение). MCP регистрируются при старте.

Откат

Полностью удалить регистрации:

for n in payfin-kb payfin-code gitlab atlassian; do claude mcp remove -s user "$n"; done

Для Windows PowerShell: @('payfin-kb','payfin-code','gitlab','atlassian') | %{ claude mcp remove -s user $_ }
